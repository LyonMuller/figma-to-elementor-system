Elementor MCP Contract

The plugin expects an externally configured Elementor MCP server. It does not store WordPress usernames, application passwords, MCP headers, or target-site credentials.

Expected Capabilities

  • Read global settings, pages, templates, widgets, page structure, container schema, and widget schema.
  • Read existing Global Colors, Global Fonts, Theme Style, responsive controls, and representative clean Elementor structures before mapping or building.
  • Update global colors and global typography only when the user explicitly authorizes global changes.
  • Create draft pages and templates.
  • Add and update containers and widgets.
  • Save reusable templates when supported.

Layout Contract

  • Create containers only for shared layout behavior, responsive control, semantic grouping, reuse, or native Elementor settings that cannot be applied directly to a child widget.
  • Do not mirror every Figma frame or group as an Elementor container.
  • Prefer native controls, existing globals, responsive breakpoints, and rem-capable unit controls before any exception path.
  • Record divergences when Elementor cannot express a Figma detail with native controls and existing globals.

Permission Rule

Publishing, deletion, custom code snippets, custom CSS, new globals, hardcoded style values, and destructive content replacement require explicit user authorization.
