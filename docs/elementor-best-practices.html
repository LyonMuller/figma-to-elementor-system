Elementor Best Practices

  • Read existing Elementor Global Colors, Global Fonts, Theme Style, widget schemas, container schema, and representative page/template structures before mapping or building.
  • Use only existing Global Colors and Global Fonts unless the user explicitly authorizes new globals.
  • Use Containers/Flexbox as the structural default.
  • Create containers only for shared layout behavior, responsive control, semantic grouping, reuse, or native Elementor settings that cannot be applied directly to a child widget.
  • Avoid unnecessary nested containers, one-widget wrappers, spacer containers, and Figma group mirroring.
  • Avoid fixed width or height values on containers; use responsive flow, width 100%, auto, max-width, min-height, and breakpoint controls only when needed.
  • Use native widgets and native controls before considering any custom code.
  • Do not use custom CSS, inline CSS, custom classes, IDs, or external stylesheets unless explicitly approved by the user.
  • Prefer rem units for padding, margin, gap, border radius, font size, and general spacing where Elementor controls allow unit selection.
  • Do not hardcode hex, RGBA, sampled Figma colors, invented tokens, or rigid pixel widths.
  • Keep heading semantics aligned with the page outline.
  • Use responsive controls rather than duplicating sections.
  • Keep assets optimized and avoid layout shifts.
  • Do not publish without explicit authorization.
