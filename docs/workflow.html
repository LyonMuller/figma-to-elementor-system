Workflow

  1. Receive a Figma URL, node, frame, selected design, or explicit scope.
  2. Validate Figma MCP and Elementor MCP availability.
  3. Run Figma analysis and capture design context, variables, screenshots, and metadata.
  4. Read existing Elementor globals, schemas, widgets, and representative clean page/template structures.
  5. Build the Elementor-compatible design system by mapping Figma values to existing Elementor globals.
  6. Create the Figma to Elementor mapping with the smallest maintainable container tree.
  7. Optimize assets when assets are present.
  8. Build or update Elementor drafts, templates, containers, and widgets through native controls.
  9. Adapt and validate desktop, tablet, and mobile behavior.
  10. Run QA for fidelity, existing globals, structure, CSS approval, accessibility, responsiveness, and performance.
  11. Generate final HTML documentation and reports.

Design system work and Elementor MCP observation must always happen before Elementor page or template construction.
