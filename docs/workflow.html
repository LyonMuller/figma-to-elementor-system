Workflow

  1. Receive a Figma URL, node, frame, selected design, or explicit scope.
  2. Validate Figma MCP and Elementor MCP availability.
  3. Run Figma analysis and capture design context, variables, screenshots, and metadata.
  4. Build the Elementor-compatible design system.
  5. Create the Figma to Elementor mapping.
  6. Optimize assets when assets are present.
  7. Build or update Elementor global settings, drafts, templates, containers, and widgets.
  8. Adapt and validate desktop, tablet, and mobile behavior.
  9. Run QA for fidelity, tokens, structure, accessibility, responsiveness, and performance.
  10. Generate final HTML documentation and reports.

Design system work must always happen before Elementor page or template construction.
