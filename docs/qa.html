QA Checklist

  • Figma screenshot and Elementor output visually match without sacrificing native Elementor maintainability.
  • Existing Global Colors and Global Fonts are used consistently.
  • No hardcoded hex, RGBA, sampled colors, invented tokens, or unauthorized new globals are present.
  • Containers are semantic, shallow, and justified by layout, responsive behavior, reuse, or native Elementor control needs.
  • No containers are used only as spacers, one-widget wrappers, Navigator organization, CSS substitutes, or Figma group mirrors.
  • Headings follow a coherent document outline.
  • Desktop, tablet, and mobile layouts are stable.
  • Spacing, typography, radius, and gaps use rem where Elementor controls support unit selection.
  • No rigid fixed widths or desktop-only dimensions harm responsiveness.
  • Images have appropriate alt text or decorative treatment.
  • Color contrast, tap targets, and focus states are acceptable.
  • Assets are optimized and no unnecessary scripts were added.
  • No custom CSS, inline CSS, custom classes, IDs, external stylesheets, or custom code snippets were used without explicit user approval.
  • Unmapped Figma details are recorded as divergences instead of being solved with unapproved CSS.
  • No content was published without explicit authorization.
